## Context Priming

*GATE: Read all files in this section before starting any implementation.*

**Specification**:
- `docs/specs/005-kitty-session-fix/product-requirements.md` — PRD, acceptance criteria
- `docs/specs/005-kitty-session-fix/solution-design.md` — SDD, ADRs, interface specs, implementation examples

**Key Design Decisions**:
- **ADR-1**: `/dev/tty` key reading — use `open('/dev/tty', 'rb')` instead of `sys.stdin` in `which_key.py`; bypasses overlay stdin-is-not-a-TTY failure.
- **ADR-2**: fzf picker — `session-picker.sh` uses `/opt/homebrew/bin/fzf` for interactive session selection; launched as a new overlay by `which_key.py`.
- **ADR-3**: Shell save overlay — `session-save.sh` uses `zsh read` for the name prompt; no Python TTY manipulation.
- **ADR-4**: Remove `session_name` line — `save_session.py:build_session()` must NOT write `session_name {name}`; this command is invalid and causes `ValueError` on every load.
- **ADR-5**: Two hints only — HINTS = `[("w", "session picker", "pick"), ("s", "save session", "save")]`; the `l` (last session) binding is removed.

**Critical bug context**:
- Root cause of load failure: `which_key.py` line 89 writes `session_name {name}` to every saved session file. This command does not exist in kitty's session syntax. Kitty throws `ValueError: Unknown command in session file: session_name` on load.
- Root cause of menu disappearing: `read_one_key()` calls `termios.tcgetattr(sys.stdin.fileno())` — in overlay context stdin is not a TTY, raising `OSError` which the broad `except Exception` silently swallows.
- `set-tab-color` reset bug: current code calls `_rc(listen_on, "set-tab-color", "--reset")` — `--reset` flag does not exist. Correct syntax: `_rc(listen_on, "set-tab-color", "active_bg=NONE")`.

### Phase 2: Core Fixes — which_key.py

Fixes the two critical bugs in the existing kitten that cause the hint overlay to silently vanish and the tab colour reset to be a no-op. Phase 2 tasks T2.1 and T2.2 can be done in parallel since they touch different functions.

- [ ] **T2.1 Fix read_one_key() — /dev/tty input** `[activity: scripting]` `[parallel: true]`

  1. Prime: Read `which_key.py:read_one_key()` lines 152-159; read SDD ADR-1 and Implementation Example "which_key.py — /dev/tty key reading" `[ref: SDD/ADR-1; SDD/Implementation Examples]`
  2. Test: Run `kitten @ --to unix:/tmp/kitty launch --type=overlay python3 ~/.config/kitty/which_key.py` — overlay appears then immediately vanishes (expected: red baseline, confirms the stdin bug is present)
  3. Implement: Replace `which_key.py:read_one_key()` with the `/dev/tty` version from the SDD example. Change `sys.stdin` to `open('/dev/tty', 'rb')`:
     ```python
     def read_one_key():
         with open('/dev/tty', 'rb') as tty_fd:
             fd = tty_fd.fileno()
             old = termios.tcgetattr(fd)
             try:
                 tty.setraw(fd)
                 return tty_fd.read(1).decode('utf-8', errors='replace')
             finally:
                 termios.tcsetattr(fd, termios.TCSADRAIN, old)
     ```
  4. Validate: Re-run overlay — overlay now stays open, renders hint box, responds to keypress
  5. Success:
     - [ ] Overlay renders and stays visible `[ref: PRD/Feature 5 AC]`
     - [ ] Pressing Escape closes overlay cleanly `[ref: PRD/Feature 5 AC]`

- [ ] **T2.2 Fix set_tab_color() reset syntax** `[activity: scripting]` `[parallel: true]`

  1. Prime: Read `which_key.py:set_tab_color()` lines 41-45; read SDD Implementation Example "set-tab-color reset" `[ref: SDD/kitten @ Remote Control — Verified API; SDD/Implementation Examples]`
  2. Test: Trigger the current reset call: `kitten @ --to unix:/tmp/kitty set-tab-color --reset` in a terminal — expect error (confirms `--reset` doesn't exist)
  3. Implement: In `set_tab_color()`, change the `else` branch from `_rc(listen_on, "set-tab-color", "--reset")` to `_rc(listen_on, "set-tab-color", "active_bg=NONE")`
  4. Validate: `kitten @ --to unix:/tmp/kitty set-tab-color "active_bg=NONE"` runs without error; active tab colour returns to theme default
  5. Success: Tab colour resets to theme default when overlay closes `[ref: SDD/ADR-4; PRD/Feature 5 AC]`
