Extend the existing bin/wez-session bash script to add four session subcommands alongside the existing daemon subcommand group. Preserve ALL existing daemon code unchanged -- only add new code.

**New constants** (add after existing constants):
```
SESSIONS_DIR="$HOME/.config/wezterm/sessions"
```

**New helper functions:**

1. `ensure_mux_running()` -- Check that mux socket is active before session commands. If not, print error: "Error: WezTerm mux daemon is not running. Run 'wez-session daemon start' first." and exit 1.

2. `get_active_workspace()` -- Use `wezterm cli list --format json` to extract the current workspace name. Parse with a simple approach: pipe JSON through `python3 -c "import json,sys; data=json.load(sys.stdin); print(data[0]['workspace'] if data else 'default')"` (python3 is always available on macOS). Fall back to "default" if no panes exist.

3. `get_active_workspaces()` -- Use `wezterm cli list --format json` to extract unique workspace names as a newline-separated list. Use python3 to parse: extract unique workspace values from the JSON array.

4. `relative_time()` -- Accept a unix timestamp, compute difference from now, return human-friendly string: "just now" (<60s), "N min ago" (<3600s), "N hours ago" (<86400s), "N days ago" (>=86400s). Use `date +%s` for current time.

**Session subcommand functions:**

5. `session_create()` -- Accepts $1 as session name.
   - Validate name is not empty: "Error: Session name required. Usage: wez-session create <name>"
   - Validate name matches ^[a-zA-Z0-9_-]+$: "Error: Session name must contain only letters, numbers, dashes, and underscores"
   - Call `ensure_mux_running`
   - Check if workspace already exists (via get_active_workspaces). If so: "Session '<name>' already exists." and exit 0.
   - Run: `wezterm cli spawn --workspace "$name" --cwd "$HOME"`
   - Print: "Created session: <name>"

6. `session_list()` -- No arguments.
   - Call `ensure_mux_running` (soft -- if daemon not running, still show saved sessions)
   - Collect active workspaces from `wezterm cli list --format json` (may fail if daemon down)
   - Collect saved sessions from `ls sessions/*.json`
   - Merge: active workspaces + saved-only sessions (deduplicate by name)
   - Get current workspace from get_active_workspace
   - Print aligned table with columns: NAME, STATUS, LAST SAVED
   - STATUS: "active" if workspace is running, "saved" if only JSON exists
   - NAME: prefix with "* " if name matches current workspace (git branch style)
   - LAST SAVED: relative timestamp from JSON's last_saved field, or "-" if never saved
   - Sort: current session first, then by last_saved descending, then alphabetically
   - Read last_saved from JSON files using python3: `python3 -c "import json; data=json.load(open('file.json')); print(data.get('last_saved', ''))"`
   - If no sessions exist at all, print: "No sessions found."
   - Output has NO color (pipe-friendly per user decision)

7. `session_save()` -- Accepts optional $1 as session name.
   - Call `ensure_mux_running`
   - If no name given, use `get_active_workspace()` to determine current workspace name
   - Validate workspace exists (is it in active workspaces list?). If not: "Error: Workspace '<name>' is not active. Can only save running workspaces."
   - Write the save logic directly in bash. Use `wezterm cli list --format json` to capture workspace state (panes, CWDs, processes), then write JSON to sessions/<name>.json in the same format state.lua uses. This keeps the CLI self-contained and avoids complex Lua IPC.
   - Implementation of bash-side save:
     a. Get all panes for the target workspace from `wezterm cli list --format json`
     b. Group panes by tab (tab_id)
     c. For each pane, extract: cwd, title (as process proxy)
     d. Build JSON using python3:
     ```
     python3 -c "
     import json, sys, time
     data = json.load(sys.stdin)
     workspace = '$name'
     panes = [p for p in data if p['workspace'] == workspace]
     tabs = {}
     for p in panes:
         tid = p['tab_id']
         if tid not in tabs:
             tabs[tid] = {'title': '', 'active': False, 'panes': []}
         tabs[tid]['panes'].append({
             'index': len(tabs[tid]['panes']),
             'is_active': p.get('is_active_pane', False),
             'cwd': p.get('cwd', ''),
             'process': p.get('title', '').split(' ')[0] if p.get('title') else None,
             'left': 0, 'top': 0,
             'width': p.get('size', {}).get('cols', 80) if isinstance(p.get('size'), dict) else 80,
             'height': p.get('size', {}).get('rows', 24) if isinstance(p.get('size'), dict) else 24,
         })
         if p.get('is_active_pane'):
             tabs[tid]['active'] = True
     result = {
         'version': 1,
         'workspace': workspace,
         'last_saved': int(time.time()),
         'tabs': list(tabs.values()),
     }
     print(json.dumps(result, indent=2))
     "
     ```
     e. Write JSON to `$SESSIONS_DIR/<name>.json` (atomic: write to .tmp, then mv)
     f. Print: "Saved session: <name>"

8. `session_delete()` -- Accepts $1 as session name, optional -f/--force flag.
   - Parse flags: if first arg is "-f" or "--force", set force=true and shift
   - Validate name is not empty: "Error: Session name required. Usage: wez-session delete [-f] <name>"
   - Protect default: "Cannot delete 'default' -- it's the fallback workspace."
   - Validate session exists (active workspace OR saved JSON). If not: "Error: Session '<name>' not found."
   - If not force: prompt "Delete session '<name>'? (y/N) " and read response. Only proceed on "y" or "Y".
   - Remove JSON file if it exists: rm -f "$SESSIONS_DIR/<name>.json"
   - If workspace is active, close its panes: get all pane IDs for that workspace from `wezterm cli list --format json`, then for each pane: `wezterm cli send-text --pane-id <id> "exit\n"`. Then inform user which workspace they were switched to.
   - Print: "Deleted session: <name>"
   - If was active session, also print: "Switched to '<fallback>'." (determine fallback: first other active workspace, or "default")
