  <action>
Modify `lua/session/picker.lua` to add a `mode` parameter to `show_picker(window, pane, mode)` with default `mode = mode or "switch"` for backward compatibility.

**Switch mode changes (extend existing behavior):**
- After the existing "__create__" sentinel, check if any deletable sessions exist (sessions where `session.name ~= current`). If yes, append a sentinel: `{ id = "__delete_mode__", label = "🗑 Delete mode..." }`.
- In the callback, handle `id == "__delete_mode__"` by calling `M.show_picker(inner_window, inner_pane, "delete")`.

**Delete mode (new branch in the mode conditional):**
- Set `title = "Sessions [DELETE]"`.
- Build choices array: for each session from `manager.list_sessions()`:
  - If `session.name == current`: add `{ id = session.name, label = "  " .. session.name .. " (current)" }` (no trash icon, two-space indent for alignment).
  - Otherwise: add `{ id = session.name, label = "🗑 " .. session.name }`.
- Do NOT add the "__empty__" sentinel or "__create__" sentinel in delete mode.
- Append sentinel: `{ id = "__back__", label = "← Back to sessions..." }`.

**Delete mode callback handling:**
- `if id == "__back__"` then `M.show_picker(inner_window, inner_pane, "switch")` and return.
- If `id == current` (user selected current session in delete mode): silent return (no-op).
- Otherwise, show delete confirmation via `PromptInputLine`:
  - `description = "Delete '" .. id .. "'? Type 'yes' to confirm:"`
  - In the PromptInputLine callback:
    - If `not line` (Escape pressed): `M.show_picker(win, p, "delete")` (return to delete mode).
    - Normalize: `local normalized = string.lower(line)`.
    - If `normalized ~= "y" and normalized ~= "yes"`: `M.show_picker(win, p, "delete")` (cancelled, return to delete mode).
    - If confirmed: call `manager.delete_session(id)`.
      - If `not ok`: show toast `win:toast_notification("Delete Error", err or "Failed to delete session", nil, 3000)` then `M.show_picker(win, p, "delete")`.
      - If ok: check remaining sessions via `manager.list_sessions()`. Count deletable sessions (those where `s.name ~= wezterm.mux.get_active_workspace()`). If `has_deletable` is true, call `M.show_picker(win, p, "switch")`. If no deletable sessions remain, do nothing (picker auto-closes).

**Escape behavior in delete mode (Claude's discretion):** Accept InputSelector's default Escape behavior (close picker entirely). The "← Back to sessions..." sentinel is the explicit way to return to switch mode. This is simpler and avoids fighting the InputSelector API.

**Structural note:** Keep the existing switch mode code path intact. The mode parameter adds a conditional branch -- do not restructure the existing working switch logic. The `active_sessions` lookup table built at the top of `show_picker` is still needed for switch mode's restore-vs-switch decision.
