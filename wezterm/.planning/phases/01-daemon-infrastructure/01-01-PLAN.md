      - `STATE_DIR="$HOME/.local/state/wezterm"`
      - `SOCKET_PATH="$STATE_DIR/wezterm.sock"`

      - `KeepAlive`: true
      - `StandardOutPath` and `StandardErrorPath`: point to `$LOG_PATH`
      - `EnvironmentVariables.PATH`: `/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin`

  <name>Task 2: Create Lua session modules (init.lua and daemon.lua)</name>
  <files>lua/session/init.lua, lua/session/daemon.lua</files>
  <action>
    Create `lua/session/daemon.lua`:
    - Module table: `local M = {}`
    - `M.DOMAIN_NAME = "local-mux"` — constant for the unix domain name
    - `M.get_socket_path()`: Returns `wezterm.home_dir .. "/.local/state/wezterm/wezterm.sock"`
    - `M.get_unix_domain_config()`: Returns table `{ { name = M.DOMAIN_NAME, socket_path = M.get_socket_path() } }` — the value for `config.unix_domains`
    - `M.get_startup_args()`: Returns `{ "connect", M.DOMAIN_NAME }` — the value for `config.default_gui_startup_args`
    - `M.is_connected(pane)`: Returns boolean — checks `pane:get_domain_name() == M.DOMAIN_NAME`. Nil-safe: returns false if pane is nil.
    - `M.get_status(pane)`: Returns a table `{ connected = bool, domain = string }` for use by status bar. `domain` is `pane:get_domain_name()` (nil-safe, defaults to `"local"`).
    - `return M`

    Create `lua/session/init.lua`:
    - Module table: `local M = {}`
    - `local wezterm = require("wezterm")`
    - `local daemon = require("lua.session.daemon")`
    - `M.daemon = daemon` — expose daemon submodule
    - `M.apply_to_config(config, opts)`:
      - `opts` has shape `{ enabled = bool }` (defaults to `{ enabled = true }`)
      - If not `opts.enabled`, return early (do nothing)
      - Set `config.unix_domains = daemon.get_unix_domain_config()`
      - Set `config.default_gui_startup_args = daemon.get_startup_args()`
    - `return M`

    Important implementation notes:
    - Both files must use `local wezterm = require("wezterm")` — WezTerm provides this globally
    - Do NOT use `package.path` manipulation — WezTerm adds `config_dir` to the Lua path automatically for `require("lua.session.xxx")` style imports (test this in Task 1's test_lua_modules)
    - Keep modules minimal — Phase 2+ will add state.lua, picker.lua etc.
    - All functions must be nil-safe (no bare method calls on potentially nil values)
