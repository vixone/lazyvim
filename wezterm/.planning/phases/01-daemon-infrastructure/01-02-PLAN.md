    Modify `wezterm.lua` with these specific changes (preserve ALL existing code):

    ```lua

    ```

    **2. Add config flag (after line 12, after config_builder block):**
    ```lua
    -- ─── SESSION MANAGER ──────────────────────────────────────────────
    -- Set to false to disable mux daemon and use vanilla WezTerm
    local session_manager = {
      enabled = true,
    }
    ```

    **3. Apply session config (after the config flag, before THEME MODE section):**
    ```lua
    session.apply_to_config(config, session_manager)
    ```

    **4. Extend update-status handler to include mux indicator:**
    In the existing `wezterm.on("update-status", ...)` handler (around line 428), modify the hints table construction:

    BEFORE the existing hints table, add mux status check:
    ```lua
    -- Mux connection indicator
    local mux_connected = session_manager.enabled and session.daemon.is_connected(pane)
    ```

    AFTER the existing hints table is defined (after the `{ key = "..T", label = "Theme" }` entry), conditionally prepend the mux indicator:
    ```lua
    if mux_connected then
      table.insert(hints, 1, { key = "\u{f0ac}", label = "Mux" })  -- nf-fa-globe icon
    end
    ```

    Use `\u{f0ac}` (nf-fa-globe from Nerd Fonts) as the mux indicator icon -- it's visually distinct from existing hint keys which use keyboard shortcuts. Alternative: use `wezterm.nerdfonts.cod_radio_tower` if globe doesn't render well. The icon should be a single character, not a keyboard shortcut, to visually separate "status" from "action hints".

    **5. Do NOT modify any other sections.** These must remain unchanged:
    - Theme definitions and read_theme_mode function
    - Font, appearance, window, scrollback, bells, cursor, performance settings
    - All keybindings in config.keys
    - format-tab-title handler
    - copy-on-select logic in update-status
    - smart-splits configuration
    - `return config` at the end

    **Error handling approach (Claude's discretion):**
    - Wrap the `require("lua.session")` in a pcall. If it fails (module not found), set `session` to a stub that does nothing, and `session_manager.enabled` to false. This prevents config breakage if lua/session/ files are missing.
    - Pattern:
      ```lua
      local session_ok, session = pcall(require, "lua.session")
      if not session_ok then
        session = { apply_to_config = function() end, daemon = { is_connected = function() return false end } }
      end
      ```
