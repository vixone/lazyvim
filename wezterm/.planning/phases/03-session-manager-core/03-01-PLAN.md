<task type="auto">
  <name>Task 2: Create manager.lua and wire into init.lua</name>
  <files>lua/session/manager.lua, lua/session/init.lua</files>
  <action>
Create `lua/session/manager.lua` with the following functions. Use `local wezterm = require("wezterm")` and `local state = require("lua.session.state")` at the top. Follow the `local M = {} ... return M` module pattern.

**Function 1: `M.create_session(name)`**
Per user decision: idempotent -- if session already exists (running workspace OR JSON file), switch to it.
1. Validate `name`: return `nil, "Session name cannot be empty"` if nil/empty. Return `nil, "Session name must contain only alphanumeric characters, dashes, and underscores"` if `not name:match("^[%w%-_]+$")`.
2. Check if session already exists: call `M.list_sessions()`, iterate results, if any `session.name == name` then call `M.switch_session(name)` and return its result.
3. Auto-save current workspace: call `state.save_current_workspace()` (per locked decision).
4. Create new workspace: `local tab, pane, window = wezterm.mux.spawn_window({ workspace = name, cwd = wezterm.home_dir })`. CRITICAL: Do NOT pass a `domain` parameter (Pitfall #4 -- causes duplicate tabs).
5. If `window` is nil, return `nil, "Failed to create workspace"`.
6. Log with `wezterm.log_info`.
7. Return `true`.

**Function 2: `M.list_sessions()`**
Per user decisions: include default, sorted by last_saved (most recent first), show active/saved status.
1. Initialize `local sessions = {}` and `local seen = {}`.
2. Enumerate active workspaces: iterate `wezterm.mux.all_windows()`, for each `window:get_workspace()` that is not nil and not in `seen`, add to `sessions` with `{ name = workspace, active = true, last_saved = nil }` and mark in `seen`.
3. Enumerate JSON files: build `sessions_dir = wezterm.config_dir .. "/sessions"`. Use `io.popen('ls -1 "' .. sessions_dir .. '"/*.json 2>/dev/null')`. For each line, extract basename with `filename:match("([^/]+)%.json$")`. If basename not in `seen`, load metadata with `state.load_workspace(basename)` and add `{ name = basename, active = false, last_saved = layout and layout.last_saved }`. If basename IS in `seen` (active workspace), update that entry's `last_saved` from the loaded layout.
4. Sort by `last_saved` descending (most recent first). If both have timestamps, compare numerically. If only one has timestamp, it comes first. If neither, sort alphabetically by name.
5. Return `sessions`.

**Function 3: `M.switch_session(name)`**
Per user decisions: auto-save before switching.
1. Auto-save current workspace: `state.save_current_workspace()`.
2. Check if workspace already running: iterate `wezterm.mux.all_windows()` looking for `window:get_workspace() == name`. If found, set `exists = true`.
3. If not running, check if JSON exists by calling `state.load_workspace(name)`. If JSON exists, spawn into that workspace: `wezterm.mux.spawn_window({ workspace = name, cwd = wezterm.home_dir })` (no domain parameter). If JSON does not exist, return `nil, "Session '" .. name .. "' not found"`.
4. If workspace was already running, we just need to switch focus. NOTE: Programmatic workspace switching requires an action callback context (window:perform_action). Since this API may be called from event handlers that DO have window context, or from contexts that don't, use `wezterm.mux.spawn_window({ workspace = name, cwd = wezterm.home_dir })` as the reliable cross-context method -- it activates the workspace by spawning a window in it. If the workspace already has windows, WezTerm will switch to it and the new window joins that workspace.
5. Log with `wezterm.log_info`.
6. Return `true`.

IMPORTANT DESIGN NOTE: `switch_session` uses `mux.spawn_window` as the switching mechanism because there is no direct `mux.set_active_workspace()` API. This spawns a new window tab in the target workspace, which causes WezTerm to switch to that workspace. This is the verified pattern from research. In Phase 6 (picker), switching will use `window:perform_action(act.SwitchToWorkspace{name})` from keybinding context where `window` is available -- but for the programmatic API callable from any context, `spawn_window` is the correct approach.

**Function 4: `M.delete_session(name)`**
Per user decisions: full cleanup, active deletion allowed, default protected, no confirmation.
1. Protect default: if `name == "default"`, return `nil, "Cannot delete default workspace"`.
2. Validate name exists: call `M.list_sessions()`, check if any session matches. If not found, return `nil, "Session '" .. name .. "' not found"`.
3. Check if deleting active session: `local current = wezterm.mux.get_active_workspace()`. If `name == current`, find an alternative session to switch to (first non-deleted session from `list_sessions`, or "default" as fallback), then call `M.switch_session(target)`.
4. Remove JSON file: `local sanitized = name:gsub("[^%w%-_]", "_")`, build path `wezterm.config_dir .. "/sessions/" .. sanitized .. ".json"`, call `os.remove(path)`. Log warning if removal fails but continue.
5. Close all workspace windows: iterate `wezterm.mux.all_windows()`, for each window where `window:get_workspace() == name`, iterate `window:tabs()`, for each tab iterate `tab:panes()`, call `pane:kill()` on each.
6. Log with `wezterm.log_info`.
7. Return `true`.
