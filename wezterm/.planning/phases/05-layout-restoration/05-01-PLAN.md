<task type="auto">
  <name>Task 2 (Wave 1): Implement session restoration in manager.lua</name>
  <files>lua/session/manager.lua</files>
  <action>
Extend lua/session/manager.lua with restoration functions. Add these BELOW the existing delete_session function, keeping all existing code unchanged.

**1. RESTORABLE_PROCESSES allowlist (module-level, above the new functions):**
```lua
local RESTORABLE_PROCESSES = {
  nvim = "nvim",
  claude = "claude",
  npm = "npm run dev",
  node = "node",
}
-- Shells (zsh, bash, fish) are NOT in this table -- they spawn by default
```

**2. `M.attach_session(name)` -- Smart attach (public API):**
- Validate name (same pattern as create_session: non-empty, alphanumeric+dash+underscore)
- Check if workspace is already running: iterate `wezterm.mux.all_windows()`, compare `window:get_workspace() == name`
- If running: call `M.switch_session(name)` and return
- If not running: check if JSON exists via `state.load_workspace(name)` -- if nil, return nil + error
- If JSON exists: call `M.restore_session(name)` and return

**3. `M.restore_session(name)` -- Full restoration (public API):**
- Load layout via `state.load_workspace(name)` -- return nil + error if missing
- Auto-save current workspace: `state.save_current_workspace()`
- Get first tab's first pane CWD for initial workspace spawn (fall back to `wezterm.home_dir` if nil)
- Spawn new workspace: `wezterm.mux.spawn_window({ workspace = name, cwd = first_cwd })`
- Return nil + error if window creation fails
- Call `M._restore_layout(layout, window, pane)` where `pane` is the initial pane returned by spawn_window
- Log success: `wezterm.log_info("Restored session: " .. name)`
- Return true

**4. `M._restore_layout(layout, window, initial_pane)` -- Internal helper:**
- Iterate `layout.tabs` by index
- For tab_idx == 1: reuse `window:active_tab()` as target_tab, `initial_pane` as first_pane_in_tab
- For tab_idx > 1: call `window:spawn_tab({ cwd = first_pane_cwd_or_home })` -- this returns a tab AND pane. Use `local new_tab, new_pane = window:spawn_tab(...)`. The first_pane_in_tab is `new_pane`.
- Set tab title if `tab_data.title` is non-empty: `target_tab:set_title(tab_data.title)` (NOTE: WezTerm MuxTab has `set_title()` method)
- Call `M._restore_tab_panes(tab_data, first_pane_in_tab)` for each tab

**5. `M._restore_tab_panes(tab_data, first_pane)` -- Internal helper:**
- Get panes list from `tab_data.panes`
- If empty, return immediately
- Configure first pane: call `M._configure_pane(first_pane, panes[1])`
- For panes 2 through #panes:
  - Infer split direction from geometry: compare current pane's `left` vs previous pane's `left + (width / 2)`. If current `left` is greater, direction is "Right". Otherwise direction is "Bottom".
  - Get CWD for new pane: `panes[i].cwd or wezterm.home_dir`
  - Split from first_pane using `first_pane:split({ direction = direction, cwd = cwd })`. This is the Lua API for splitting (MuxPane:split method). Wrap in pcall for error isolation.
  - If split succeeds, configure the new pane: `M._configure_pane(new_pane, panes[i])`
  - If split fails, log warning: `wezterm.log_warn("Failed to split pane " .. i .. ": " .. tostring(err))`
  - If i > 4, log warning about degraded accuracy

IMPORTANT: Use `MuxPane:split()` Lua method instead of `wezterm cli split-pane`. The research suggested CLI for splitting, but MuxPane has a `:split()` method that takes `{ direction = "Right"|"Bottom"|"Left"|"Top", cwd = path }` and returns a new MuxPane synchronously. This avoids Pitfall #3 (async CLI ordering). Wrap each split call in pcall to handle failures gracefully.

If MuxPane:split() does not exist or fails, fall back to wezterm.run_child_process with "wezterm cli split-pane". Implement this as: try pcall(pane.split, pane, opts) first, if that errors with "no such method", use CLI fallback. Log which method is being used at info level on the first call.

**6. `M._configure_pane(pane, pane_data)` -- Internal helper:**
- If `pane_data.process` is nil, return (shell spawns by default)
- If process is a shell name (zsh, bash, fish, sh), return (already running)
- Check `RESTORABLE_PROCESSES[pane_data.process]`
- If found: use `pane:send_text(command .. "\n")` where command comes from the RESTORABLE_PROCESSES table value
- If not found and process is non-nil and not a shell: log info about skipped process

**Error handling throughout:**
- Wrap all spawn/split operations in pcall
- On failure: log warning, continue with remaining tabs/panes (partial restoration)
- Never crash the entire restore on a single pane failure
