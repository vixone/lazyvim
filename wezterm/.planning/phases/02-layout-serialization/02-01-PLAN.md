Create `lua/session/state.lua` following the `local M = {} ... return M` pattern established in daemon.lua.

The module must implement these functions:

**`M.uri_to_path(uri_obj)`**
- Accepts a Url object (from `pane:get_current_working_dir()`) or nil
- Returns nil if input is nil
- Converts to string with `tostring(uri_obj)` first
- Uses `wezterm.target_triple` for platform detection:
  - Windows (`target:find("windows")`): strip `file:///` prefix
  - macOS/Linux: strip `file://` + optional hostname + leading slash -- result starts with `/`
- Use the pattern: `uri_str:gsub("^file://[^/]*/", "/")` for macOS/Linux (handles both `file:///path` and `file://hostname/path`)

**`M.capture_workspace()`**
- Gets active workspace name via `wezterm.mux.get_active_workspace()`
- Iterates `wezterm.mux.all_windows()`, filters to windows matching active workspace via `window:get_workspace()`
- For each matching window, iterates `window:mux_window():tabs()` (NOTE: for mux windows, `window` IS a MuxWindow, so check if `mux_window()` method exists; if iterating `mux.all_windows()`, these are MuxWindow objects directly -- use `window:tabs()` not `window:mux_window():tabs()`)
- CORRECTION: `wezterm.mux.all_windows()` returns MuxWindow objects. Call `window:tabs()` directly.
- For each tab, calls `tab:panes_with_info()` to get pane layout
- For each pane_info:
  - `pane_info.pane:get_current_working_dir()` -- nil-safe, convert with `uri_to_path()`
  - `pane_info.pane:get_foreground_process_name()` -- nil-safe, extract basename with `process and process:match("([^/]+)$") or nil`
  - Store: `index`, `is_active`, `cwd` (string or nil), `process` (string or nil), `left`, `top`, `width`, `height`
- For each tab, store: `title` (from `tab:get_title()`), `active` (from `tab:is_active()`), `panes` array
- Returns Lua table:
  ```lua
  {
    version = 1,
    workspace = "workspace-name",
    last_saved = os.time(),  -- Unix timestamp (integer, clean JSON diff)
    tabs = { ... }
  }
  ```
- Use `os.time()` for timestamp (integer, always available) rather than `wezterm.time.now()` which may serialize as a complex object. The goal is clean, diffable JSON.

**`M.save_current_workspace()`**
- Calls `capture_workspace()` to get layout table
- If `#layout.tabs == 0`, log warning and return false (skip empty workspaces)
- Sanitize workspace name for filename: replace non-alphanumeric chars (except `-` and `_`) with `_` to avoid filesystem issues
- Build path: `wezterm.config_dir .. "/sessions/" .. sanitized_name .. ".json"`
- Ensure `sessions/` directory exists: `wezterm.run_child_process({ "mkdir", "-p", sessions_dir })`
- Write to temp file (`filename .. ".tmp"`) using `io.open(temp, "w")`
- Encode with `wezterm.json_encode(layout)` -- note this produces compact JSON. For readability, we accept this since `jq .` can pretty-print if needed, and compact JSON still diffs cleanly line-by-line if the structure is flat enough.
- Call `f:write(json)` then `f:close()`
- Atomic rename: `os.rename(temp_file, filename)`
- Log success with `wezterm.log_info()`
- Return true on success, false on failure
- Wrap file operations in error handling (check `io.open` result)

**`M.load_workspace(name)`** (bonus -- useful for Phase 5, cheap to add now)
- Opens `sessions/<name>.json`, reads content, parses with `wezterm.json_parse()`
- Returns parsed Lua table or nil on error
- This is a read-only convenience function for later phases

Do NOT:
- Use `wezterm.time.now()` for the timestamp (it produces a complex Time object, not a clean integer)
- Call string methods on nil values (always nil-check first)
- Use `window:mux_window()` on MuxWindow objects from `mux.all_windows()` -- they ARE mux windows already
- Pretty-print JSON manually -- `wezterm.json_encode()` output is sufficient
