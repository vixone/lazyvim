<task type="auto">
  <name>Task 1: Wire debounced auto-save into update-status handler</name>
  <files>wezterm.lua</files>
  <action>
Modify the existing `update-status` event handler in `wezterm.lua` (around line 440) to add debounced auto-save.

**Changes to make:**

1. Add a module-level variable before the `update-status` handler:
   ```lua
   -- ─── AUTO-SAVE STATE ──────────────────────────────────────────────
   local save_timer = nil
   ```

2. Inside the `update-status` handler, AFTER the copy-on-select block and BEFORE the hints bar block, add:
   ```lua
   -- Debounced auto-save (2 second delay)
   -- Only save if session manager is enabled
   if session_manager.enabled then
     if save_timer then
       save_timer:cancel()
     end
     save_timer = wezterm.time.call_after(2, function()
       session.state.save_current_workspace()
     end)
   end
   ```

3. Place the auto-save block between the copy-on-select section and the mux connection indicator section. This ordering ensures:
   - Copy-on-select runs first (user-facing, immediate)
   - Auto-save schedules asynchronously (non-blocking, debounced)
   - Hints bar renders last (status display)

**What NOT to change:**
- Do not modify ANY existing keybindings
- Do not modify the hints bar content or rendering
- Do not modify the copy-on-select logic
- Do not modify the theme toggle, tab lock, or any other feature
- Do not add a save keybinding (locked decision: no manual save)
- Do not change the session_manager config block

**Guard clause:** The `if session_manager.enabled` check ensures auto-save is disabled when the user sets `session_manager.enabled = false`. This follows the same pattern used for the mux connection indicator.
